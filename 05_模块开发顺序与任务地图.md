# 05 模块开发顺序与任务地图

## 5.1 总开发顺序

### 阶段1 基础骨架
1. 创建 FastAPI 项目
2. 配置数据库、迁移、Redis
3. 建立基础目录结构
4. 建立统一异常、响应、配置模块

### 阶段2 多租户与店铺入口
5. tenant / shop 模型
6. shop_code 生成规则
7. `/store/resolve` 接口
8. store_context_token 方案

### 阶段3 商品与 SKU
9. source_product 模型
10. channel_product 模型
11. product_spec / spec_value / sku 模型
12. SKU 组合生成逻辑
13. 商品列表/详情 API
14. 商家商品同步 API

### 阶段4 购物车与订单
15. cart 模型
16. channel_order / item 模型
17. 下单接口
18. 商家订单列表
19. 客户订单列表

### 阶段5 ERP/第三方接入
20. integration 模型
21. order_push_task / log 模型
22. 订单异步推送 worker
23. 推送失败重试
24. 商家手动重推

### 阶段6 运维与质量
25. 单元测试
26. 集成测试
27. 日志与审计
28. Docker 化
29. 部署脚本
30. API 文档

## 5.2 模块依赖关系

```text
tenant/shop
   ↓
store resolve
   ↓
product + sku
   ↓
cart
   ↓
order
   ↓
push task
   ↓
integration
```

## 5.3 AI 执行粒度建议

每次只让 AI 做以下其中一种：
- 一个模型组
- 一个 service
- 一个 API
- 一个任务 worker
- 一个测试文件

不要让 AI 一次完成“整个商城系统”。
