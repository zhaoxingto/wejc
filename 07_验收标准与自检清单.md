# 07 验收标准与自检清单

## 7.1 架构验收

- [ ] 是否遵守 API -> Service -> Repository -> DB 分层
- [ ] 是否未在 router 中直接写 ORM 业务查询
- [ ] 是否模型、schema、service 分离
- [ ] 是否未把多个领域胡乱塞在一个文件里

## 7.2 多租户验收

- [ ] 业务表是否带 tenant_id / shop_id
- [ ] 查询是否基于当前店铺上下文过滤
- [ ] 是否避免通过手工参数跨店铺访问数据

## 7.3 商品验收

- [ ] 是否存在 source_product
- [ ] 是否存在 channel_product
- [ ] 是否存在 product_sku
- [ ] SKU 是否支持多属性组合
- [ ] 订单项是否保留 SKU 快照信息

## 7.4 订单验收

- [ ] 是否无支付流程
- [ ] 是否下单即生成订单
- [ ] 是否异步创建推送任务
- [ ] 推送失败是否可重试
- [ ] 是否有日志记录

## 7.5 可维护性验收

- [ ] 是否有类型标注
- [ ] 是否有 schema
- [ ] 是否有单元测试建议
- [ ] 是否有清晰文件路径
- [ ] 是否说明了影响范围

## 7.6 AI 完成后必须回答的 8 个问题

1. 这次改了哪些文件？
2. 改了哪些模型？
3. 改了哪些接口？
4. 是否修改数据库结构？
5. 是否新增迁移？
6. 是否影响多租户逻辑？
7. 如何测试？
8. 下一步最自然的任务是什么？
