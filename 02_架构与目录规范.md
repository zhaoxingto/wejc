# 02 架构与目录规范

## 2.1 推荐技术栈

后端：
- FastAPI
- SQLAlchemy 2.x
- PostgreSQL
- Redis
- Alembic
- Pydantic v2
- httpx
- Celery / Arq / RQ（任选一种异步任务框架）

前端后台：
- Vue 3
- TypeScript
- Pinia
- Element Plus 或 Naive UI

小程序：
- uni-app 或原生微信小程序

## 2.3 禁止的坏结构

AI 禁止生成以下结构：

### 坏结构1：所有逻辑都写在 router 里
```python
@router.post("/create")
def create():
    # 一堆校验
    # 一堆SQL
    # 一堆外部请求
```

### 坏结构2：所有模型写在一个文件
禁止把 tenant、shop、product、order 全写到 `models.py`

### 坏结构3：Service 直接返回 ORM 原对象给外部
应通过 schema / DTO 输出

## 2.4 分层职责

### Router/API
- 参数接收
- 基础鉴权
- 调用 service
- 返回 schema

### Service
- 核心业务规则
- 跨仓储组合
- 事务控制
- 调用外部系统任务

### Repository
- 单一实体查询/写入
- 封装 ORM 操作
- 不写业务规则

### Model
- 表结构定义
- 关系定义
- 索引定义

## 2.5 命名规范

- 表名：snake_case
- 字段名：snake_case
- API 路由：kebab 或 snake 二选一，项目内统一
- Service 类：`XxxService`
- Repository 类：`XxxRepository`

## 2.6 配置规范

必须区分：
- 本地开发
- 测试
- 预发布
- 生产

敏感信息：
- 不得硬编码
- 必须从环境变量读取
- app_secret / api_key / DB 密码必须脱离源码管理

## 2.7 事务规则

下单等关键流程必须在服务层显式管理事务。  
**外部系统推送不属于同一个主事务**，应通过任务表+异步执行完成。
