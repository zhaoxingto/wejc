# 08 禁止事项与跑偏纠正手册

## 8.1 明确禁止

### 禁止1：把系统设计成淘宝式平台商城
如果 AI 输出“首页推荐多个商家”“平台搜索商家”，默认跑偏。

### 禁止2：把 ERP 商品直接当作前台销售商品
正确是：
`source_product -> channel_product`

### 禁止3：SKU 简化成字符串
正确是：
`product_spec + product_spec_value + product_sku`

### 禁止4：下单时同步推送 ERP
正确是：
`create order -> create push task -> async worker`

### 禁止5：前端直接传任意 shop_id 并信任
正确是：
`resolve -> store_context_token`

### 禁止6：在 router 里写大段业务逻辑
必须下沉到 service。

### 禁止7：为了省事不写迁移/不写 schema
这是不可接受的。

## 8.2 常见跑偏信号

- “我先简化一下，后面再优化”
- “先用一个 product 表顶着”
- “先直接推 ERP，后面再改成任务”
- “shop_id 就前端传一下好了”
- “规格先用 JSON 文本就行”

以上都说明 AI 在偏离蓝图。

## 8.3 修正话术模板

```text
停止当前输出。
你正在偏离本项目蓝图。
请重新遵守以下硬约束：
1. 私域渠道商城，不是平台商城
2. source_product / channel_product 双层模型
3. SKU 结构化建模
4. 创建订单与外部推送解耦
5. API 严格分层
请从数据模型开始重新设计。
```
